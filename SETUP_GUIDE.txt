1. WHAT IS NEUROSHIELD?
--------------------------------------------------------------------------------

NeuroShield is a web application that helps developers find security problems
in source code. When you upload code, it:

  1. Scans source files for OWASP Top 10 vulnerabilities (Bandit + Semgrep)
  2. Checks dependencies (requirements.txt, package.json) against CVE databases
  3. Uses OpenAI GPT-4o-mini to explain each issue in plain English
  4. Suggests code fixes and generates a downloadable PDF security report

It is built as:
  - Backend  → Python + FastAPI  (runs on port 8000)
  - Frontend → React + Vite      (runs on port 5173)

3. WHAT YOU NEED TO INSTALL FIRST (one-time)
--------------------------------------------------------------------------------

Install these on your PC before running the project:

  [Required]
  • Python 3.10 or newer
    Download: https://www.python.org/downloads/
    During install, CHECK "Add Python to PATH"

  • Node.js 18 or newer (includes npm)
    Download: https://nodejs.org/

  [Required for AI explanations]
  • OpenAI API key
    Get one at: https://platform.openai.com/api-keys
    (Without this, SAST and CVE results still work — only AI text is skipped)

  [Optional but recommended]
  • Semgrep CLI (extra multi-language scanning)
    Install: pip install semgrep
    Or see: https://semgrep.dev/docs/getting-started/

4. HOW TO RUN THE PROJECT (Windows — step by step)
--------------------------------------------------------------------------------

You need TWO terminal windows open at the same time.
Use PowerShell or Command Prompt.

--- TERMINAL 1: Start the Backend ---

  cd path\to\Neuroshield\backend

  python -m venv .venv

  .venv\Scripts\activate

  pip install -r requirements.txt

  copy .env.example .env

  Now open .env in Notepad and set your OpenAI key:
    OPENAI_API_KEY=sk-your-actual-key-here

  uvicorn app.main:app --port 8000

  ✓ Backend is running when you see:
    "Uvicorn running on http://127.0.0.1:8000"

  Test it: open http://127.0.0.1:8000/docs in browser (API documentation)


--- TERMINAL 2: Start the Frontend ---

  cd path\to\Neuroshield\frontend

  npm install

  npm run dev

  ✓ Frontend is running when you see:
    "Local: http://localhost:5173/"


--- OPEN THE APP ---

  Go to:  http://localhost:5173

  (Do NOT use port 8000 for the UI — that is only the API backend)

6. HOW TO TEST THE APP (demo walkthrough)
--------------------------------------------------------------------------------

Step 1 — Login
  Use the admin account above, or register a new account.

Step 2 — Go to "Scan" (upload page)

Step 3 — Upload test code
  Use the included sample project:

    Folder:  samples\vulnerable_app\

  BEST TEST: Zip the vulnerable_app folder and upload the .zip file
             (includes both app.py AND requirements.txt)

  QUICK TEST: Upload only samples\vulnerable_app\app.py

Step 4 — Wait on the Analysis screen
  The app runs: SAST scan → CVE scan → AI explanations

Step 5 — View Results Dashboard
  You will see vulnerability cards with severity colours:
    Critical = Red    High = Orange    Medium = Yellow    Low = Green

Step 6 — Download PDF Report
  Click "Download PDF Report" on the results page.

  samples\vulnerable_app\app.py has intentional security flaws:

    • Weak random number generation

9. COMMON PROBLEMS & FIXES
--------------------------------------------------------------------------------

Problem: Browser shows {"detail":"Not Found"} at http://127.0.0.1:8000
Fix:     That is the API only. Open http://localhost:5173 for the web app.

Problem: "pip" or "python" not recognized
Fix:     Reinstall Python and check "Add Python to PATH", then restart terminal.

Problem: "npm" not recognized
Fix:     Install Node.js from nodejs.org and restart terminal.

Problem: Admin login does not work
Fix:     Delete backend\neuroshield.db and restart the backend server.
         The admin account will be recreated automatically.

Problem: No AI explanations shown
Fix:     Add a valid OPENAI_API_KEY in backend\.env and restart backend.

Problem: Zero vulnerabilities for every file (including samples)
Fix:     1) Make sure backend venv is active, then run:
           pip install -r requirements.txt
         2) Restart backend from the backend folder:
           cd backend
           .venv\Scripts\activate
           uvicorn app.main:app --port 8000
         3) Upload samples\vulnerable_app\app.py (should find ~4 issues)
         4) Check the yellow/red warning box on results page for errors
         Common cause: Bandit was not installed or not found on Windows.

Problem: Semgrep findings not showing for non-Python files (JS, Go, Java, PHP, Dockerfile)
Fix:     Make sure Semgrep is installed in your virtual environment by running:
         pip install -r requirements.txt
         Semgrep is required to analyze all non-Python source code.

Problem: KeyboardInterrupt or standard streams warning on scan results page
Fix:     Do not use the "--reload" flag when starting the backend. On Windows,
         writing uploads or PDF reports will trigger an auto-reload and interrupt
         running scans. Run the server using: uvicorn app.main:app --port 8000

Problem: Port already in use
Fix:     Close other apps using port 8000 or 5173, or change the port number.
